# Workshop Automation — T-Rex Runner with Playwright MCP

**6 automation exercises** using **Playwright MCP** + **Custom Agents** + **Accessibility-Based Locators**

---

## What You Will Build

Automated Playwright tests for every key behaviour of the T-Rex Runner game — using a live browser controlled by Copilot Agents via the Playwright MCP server. All locators come from the real accessibility tree, not guessed selectors.

---

## Prerequisites

- VS Code with GitHub Copilot (Agent Mode enabled)
- Playwright MCP installed and running (see Exercise 01, Step 1)
- Both game servers running (`http-server` port 8080 + Node.js API port 3000)
- `trex-runner/` dependencies installed (`npm install` in `trex-runner/` and `trex-runner/api/`)

---

## Showing the Browser Inside VS Code

Every exercise includes this step. Two options:

| Option | How |
|---|---|
| **VS Code Quick Open Browser Tab** | `Ctrl+Shift+P` → **Quick Open Browser Tab** → enter `http://127.0.0.1:8080` → click share icon → Allow |
| **Headed MCP browser** | Add `"--headed"` to playwright args in `.vscode/mcp.json` → restart VS Code |

## Skill File (in `.github/skills/trex-automation/`)

`SKILL.md` — Contains the full automation reference: accessibility tree, `disableCollision` pattern, score assertion patterns, locator rules, and API endpoints. All agents load this skill before generating tests.

## Run the Full Automation Suite

```bash
cd trex-runner
npx playwright test tests/trex-visual.spec.ts tests/trex-gameplay.spec.ts tests/trex-collision.spec.ts --reporter=list
npx playwright show-report
```
